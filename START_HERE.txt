🎯 API ENDPOINTS
═══════════════════════════════════════════════════════════════════════════

Authentication Routes (all at /api/auth):

  POST /register
    → Create new account
    ← Returns token & user data

  POST /login
    → Login with email/password
    ← Returns token & user data

  POST /verify-token
    → Verify JWT token validity
    ← Returns user data if valid

  GET /profile
    → Get current user profile
    ← Returns user data

  PUT /profile
    → Update user profile
    ← Returns updated user data

  POST /logout
    → Logout (client-side token removal)
    ← Returns success message

⚙️ TECHNICAL DETAILS
═══════════════════════════════════════════════════════════════════════════

Backend Stack:
  • Node.js + Express
  • JWT (jsonwebtoken)
  • Bcryptjs for password hashing
  • JSON file storage (users-data.json)
  • CORS enabled

Frontend Stack:

  • localStorage for token storage
  • Fetch API for HTTP requests
  • Modern CSS with animations

Data Storage:
  • users-data.json format:
    {
      "userId": {
        "id": "timestamp",
        "email": "user@example.com",
        "username": "username",
        "password": "$2a$10$...", // bcrypt hash
        "avatar": null,
        "birthday": "DD/MM",
        "createdAt": "ISO8601",
        "emailVerified": false
      }

🆘 TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════

Problem: "Cannot find module"
Solution: npm install

Problem: "Port 4000/3000 already in use"
Solution: Change PORT in .env or kill process

Problem: "Email already registered"
Solution: Use different email or clear users-data.json

Problem: "Invalid token"
Solution: Login again, check JWT_SECRET in .env

Problem: CORS errors
Solution: Check backend is running on port 4000

See QUICK_START.md for more troubleshooting tips!
